# 三次元測定 処理前 台車組み出力ツール (Excel VBA)

「プロセスマネージャー」（`oracledb` + TNS で Oracle に直接接続する Python アプリ）が
参照しているのと**同じビュー／テーブル**を Excel VBA から **ADODB で直接参照**し、
**三次元測定の「処理前（到着済・未処理 / `STEP='100'`）」の指図**を抽出して
**台車組み**を行い、`出力` シートへ書き出します。

## 抽出・台車組みの仕様

| 項目 | 内容 |
|------|------|
| 対象 | `MES_LOT_INFORMATION` 上で現工程＝三次元測定（`PROCESS_NAME LIKE '%三次元測定%'`）かつ `STEP='100'`（処理前）、`STATE_FLG` が完成`E`/廃棄`S`でない指図 |
| プラント | 伊勢原 `5501`（`設定` シートで変更可） |
| 並べ替え | 三次元測定の **着手予定日（次工程開始日）が早い順** |
| 台車組み | **同一品番（DB図番）**で **1台車2台まで**。早い順に2台ずつ組む（端数は1台） |
| 対象品番 | `対象品番` シートの A 列から読み込み（**後から行を追加するだけで対象が増えます**） |

### 出力列（`出力` シート）

`台車No` / `指図` / `DB図番` / `現工程` / `次工程開始日` / `品目コード` / `フロー名称` / `前工程` / `前工程着手状況`

- `次工程開始日` … 三次元測定の着手予定日（= 並べ替えキー）
- `前工程` … 工順で三次元測定の **1つ手前** の工程（`V_ASP_生産計画一覧` を `LAG()` で取得）
- `前工程着手状況` … その前工程行の `ステータス`
- `台車No` … 生成順（＝着手予定日が早い順）。同じ `台車No` が同一台車に積む2台

## 参照テーブル／ビュー（プロセスマネージャーと同じ）

- `MES_LOT_INFORMATION`（現在工程・STEP・状態）
- `MES_MASTER_PROCESS_LC`（工程名）/ `MES_MASTER_FLOW_LC`（フロー名称）
- `NHK_MASTER_RECEIVE_PRODUCT`（`OLD_PRODUCT_CODE` = DB図番）
- `V_ASP_生産計画一覧`（工程ごとの着手予定日・工順・ステータス）

SQL の中身は [`三次元測定_処理前.sql`](./三次元測定_処理前.sql) を参照（VBA が動的に同等の SQL を組み立てます）。

## セットアップ手順

### 1. 前提（クライアント環境）
- **Oracle Provider for OLE DB（`OraOLEDB.Oracle`）** がインストール済みであること。
  - **Excel と同じビット数**（32bit / 64bit）の Oracle クライアントを入れてください。
  - TNS 別名を使う場合は `tnsnames.ora` が解決できる状態にしておきます
    （プロセスマネージャーの設定では
    `\\sflise02\伊勢原２\接合・セラミック部\業務課\2.業務G\Apps\ODBtns\tnsnames.ora`）。
  - TNS を使わず **EZConnect**（`ホスト:1521/サービス名`）でも接続できます。

### 2. モジュールの取り込み（コピー＆貼り付け）

リポジトリには **UTF-8 の `M_SanjigenDaisha.bas` のみ**を置いています。
日本語環境の「ファイル > ファイルのインポート」は `.bas` を Shift-JIS として読み込み
**文字化け**するため、**コピー＆貼り付けで取り込んでください**（Unicode を保持＝化けません）。

1. `Alt+F11` で VBE を開き、「**挿入 > 標準モジュール**」。`F4`（プロパティ）で
   `(オブジェクト名)` を `M_SanjigenDaisha` に設定。
2. GitHub で `M_SanjigenDaisha.bas` を開き、**全文をコピー**（Web表示のコピペでUTF-8のまま取れます）。
3. コードペインに貼り付け。**1行目の `Attribute VB_Name = "..."` だけは貼らない**でください
   （貼ると `Option Explicit` でエラーになります）。

> 既に文字化けして取り込んだ場合は、そのモジュールを右クリック →
> 「**M_SanjigenDaisha の解放**」で削除（エクスポートは「いいえ」）してから上記でやり直し。
>
> どうしても「ファイル > ファイルのインポート」を使いたい場合は **Shift-JIS 版が必要**です。
> 必要なら言ってください、Shift-JIS 版（CRLF）を生成します。

### 3. シートの準備
初回に `実行_三次元測定台車組み` を実行すると、以下のシートが無ければ自動作成されます。

- **`設定`** … 接続情報などを入力（A列＝項目、B列＝値）

  | 項目 | 初期値 | 説明 |
  |------|-----------|------|
  | プロバイダ | `OraOLEDB.Oracle` | OLE DB プロバイダ名 |
  | データソース | `LV2_PROD` | TNS別名（既存マクロと同一。`host:1521/SERVICE` でも可）|
  | ユーザーID | `LV23_NHK` | DB ユーザー（既存マクロと同一）|
  | パスワード | `LV23_NHK` | DB パスワード（既存マクロと同一）|
  | スキーマ接頭辞 | 空欄 | 接続ユーザー＝`LV23_NHK` のため無修飾でOK。必要時のみ `LV23_NHK.` |
  | 工程名キーワード | `三次元測定` | 部分一致のキーワード |
  | プラント | `5501` | 伊勢原=5501 / 宮田=5401 |

  > 接続情報は既存の本番マクロ（`指図⇒シリアル` 等）と同じ値を**初期投入済み**です。
  > そのまま実行できます（変更が必要なときだけ書き換え）。

- **`対象品番`** … A 列（2行目以降）に DB図番。初回は依頼の 15 件を投入済み
  （`DB11048, DB13039, DB11041, DB23061, DB17114, DB17052, DB21088, DB18118,
  DB20025, DB20084, DB10015, DB12042, DB18085, DB03185, DB24009`）。
  **追加は行を足すだけ**。

- **`出力`** … 実行結果（毎回クリアして書き直し）。

### 4. 実行
- マクロ `実行_三次元測定台車組み`（英名 `RunSanjigenDaisha`）を実行。
- `開発` タブ → 図形にこのマクロを割り当てると「実行ボタン」として使えます。

## ログ（イミディエイト ウィンドウ）

VBE で **`Ctrl+G`**（表示 → イミディエイト ウィンドウ）を開いた状態で実行すると、
処理状況がリアルタイムで出力されます。出力内容:

- 設定値（**パスワードはマスク**）／対象品番の一覧
- **生成 SQL**（そのまま SQL Developer 等に貼って検証可能）
- 接続文字列（パスワードはマスク）・接続/実行/取得の各**所要時間**
- 抽出件数・先頭5件の中身・台車割当（満載/端数）・完了サマリ
- エラー時は `Err.Number` / `Err.Description` と**失敗した SQL**

出力の ON/OFF はモジュール先頭の定数 `DEBUG_LOG`（既定 `True`）で切替できます。

## 注意 / トラブルシュート

- **`プロバイダーが見つかりません`** … `OraOLEDB.Oracle` 未導入、または Excel とビット数不一致。
  正しいビット数の Oracle クライアントを導入してください。
- **接続できない / TNS が解決できない** … `データソース` を EZConnect（`host:1521/SERVICE`）で試す、
  または `tnsnames.ora`（`TNS_ADMIN`）の設定を確認。
- **該当0件** … `工程名キーワード`（既定 `三次元測定`）が実際の `工程名` 表記と一致しているか確認。
  表記が異なる場合は `設定` の `工程名キーワード` を実際の表記に変更してください。
- **`次工程開始日` が空欄** … その指図に三次元測定の生産計画行が無い場合（MESには到着済だが
  `V_ASP_生産計画一覧` 側に該当行が無い）。MES側で抽出された指図は計画が無くても表示します。
- **重いビュー対策** … `CommandTimeout` は 300 秒に設定済み（`CMD_TIMEOUT`）。

## 既存マクロと1モジュールに統合する場合

本ツールは、既存の業務マクロ（`指図⇒シリアル` / `シリアル⇒指図` / `ClearLotID` /
`ClearSerialNo` / `SplitListIntoChunks` / `CollectionToArray`）と**同じ標準モジュールに
同居できます**（プロシージャ名・定数名の重複なしを確認済み）。既存モジュールへ本ツールを
**追記**する手順は次のとおり。

1. **`Option Explicit` は先頭1回だけ** … 本ツールの `Option Explicit` 行は貼らない
   （既存モジュール先頭のものを使う）。
2. **`Private Const ～` ブロックを宣言部の先頭へ** … `M_SanjigenDaisha.bas` 冒頭の
   `Private Const SHEET_SETTING ...` 〜 `Private Const DEBUG_LOG ...` までを、
   既存モジュールの `Option Explicit` 直下（最初の `Sub` より上）へ貼り付け。
   ※ モジュールレベル定数は宣言部にしか置けないため。
3. **残りのプロシージャを末尾へ** … `Public Sub 実行_三次元測定台車組み()` から
   ファイル末尾までを、既存モジュールの一番下へ貼り付け。
4. **単独の `M_SanjigenDaisha` モジュールは削除**（同名プロシージャ重複によるコンパイル
   エラーを避けるため）。

> シートも独立（既存=`入力用` 等／本ツール=`設定`・`対象品番`・`出力`）で干渉しません。
> 接続情報は両方とも `OraOLEDB.Oracle` / `LV2_PROD` / `LV23_NHK` で共通です。
> 名前重複チェック：既存6個と本ツールの全プロシージャ・定数は重複しません。

## 「処理前」の定義について

本ツールは「**三次元測定に到着済・未処理（`STEP='100'`）**」を「処理前」と定義しています
（＝指図が三次元測定工程に到達済みで、まだ測定を始めていない状態）。
もし「**次工程が三次元測定**（まだ前工程にいる）」や「**三次元測定が未完了の全指図**」を
対象にしたい場合は、`BuildSql`（`M_SanjigenDaisha.bas`）の `WHERE` 条件を調整してください。
